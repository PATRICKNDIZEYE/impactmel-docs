## What is ImpactMEL?

**ImpactMEL** is a Monitoring, Evaluation, and Learning (M&E) platform for development organizations, NGOs, and donors. It helps teams collect information, follow progress, manage programs, and prepare donor reports in one place.

> The goal is simple: enter information once, then use it across dashboards, forms, and reports.

## Who Uses ImpactMEL?

ImpactMEL is designed for teams with different responsibilities. Each person uses the platform in a different way:

| Role | Who they are | What they do in ImpactMEL |
|---|---|---|
| **Org Admin** | Country Director, Operations Manager | Manages the organization, invites team members, configures settings, creates programs |
| **M&E Officer** | MEAL Officer, Data Manager | Defines indicators and results frameworks, manages reporting periods, reviews and approves submitted data |
| **Program Manager** | Project Manager, Technical Lead | Monitors project progress, views dashboards, manages activities |
| **Reporter** | Field Officer, Data Entry Staff | Submits indicator values for their assigned projects and periods |
| **Viewer** | Senior Management, Partner Organization | Read-only access to dashboards and reports — no data entry |
| **Donor** | Funder, Bilateral Agency | Receives shared public report links — no account required |

## Key Concepts

Before using ImpactMEL, it helps to understand how the workspace is organized. Everything in ImpactMEL follows a clear hierarchy:

### Organization
Your organization is the top-level space. All programs, projects, indicators, and reports belong to it. If you work with partner organizations, each partner has its own separate space.

### Program
A **Program** is a funded initiative — typically tied to a specific donor agreement or multi-year strategy. For example: *"WASH for All" (2025–2027)*. A program contains one or more projects and is linked to one or more donors.

### Project
A **Project** is where the day-to-day work happens within a program. It has its own budget, dates, location, and team. For example: *"Rwanda WASH Phase 1"*. Projects are where information is collected and reported.

### Results Framework
The **Results Framework** shows the path from your daily activities to long-term change:

```
Impact  →  Outcome  →  Output  →  Activities
```

Every indicator and activity is linked to part of this framework, so it is easier to see what work is contributing to which result.

### Indicator
An **Indicator** is a measure of progress toward a result. For example: *"Number of households with access to clean water"*. Indicators are created once and reused across projects. Each project sets its own target and reports values for each period.

### Reporting Period
A **Reporting Period** is a defined time window for data submission — typically quarterly or annually. For example: *"Q1 2026 (Jan–Mar)"*. Reporters submit their indicator values against a specific period.

### Report
A **Report** brings together indicator data, written updates, budget summaries, and activities for a given period. Reports can be shared by link or downloaded as a PDF.

## Data Flow

Information usually moves through ImpactMEL in this order:

```
Field Staff → Form Submission or Indicator Report
           → M&E Officer Review & Approval
           → Report Assembly
           → Donor PDF or Public Link
```

Once information is reviewed and approved, it can be used in reports without being entered again.

## Glossary

| Term | Definition |
|---|---|
| **Disaggregation** | Splitting one number into smaller groups, such as sex, age, or location |
| **Aggregation Method** | The rule used to combine values across reporting periods |
| **Baseline** | The starting value of an indicator before the project began |
| **Target** | The planned value to achieve by a given period or end of project |
| **Progress Score** | A percentage showing how close the current value is to the target |
| **Formula Indicator** | An indicator that is calculated from other indicators |
| **Share Link** | A link that lets someone read a report without logging in |
| **Roles & Permissions** | The rules that decide what each person can see and do |
| **Org Admin** | The role with the broadest access — manages members, settings, and organization data |
| **M&E Officer** | Can manage indicators, approve data, and manage reporting periods |
| **Reporter** | Can submit data for assigned projects |
| **Viewer** | Read-only access to dashboards and reports |

## What gets easier

| Task | Without ImpactMEL | With ImpactMEL |
|---|---|---|
| Tracking indicators | Spreadsheets go out of date quickly | Everyone works from the same current figures |
| Preparing donor reports | Manual copy-paste and formatting takes time | Build reports from approved information |
| Collecting field information | Separate forms create cleanup work later | Forms feed directly into reporting work |
| Sharing progress | Email attachments cause version confusion | Share a link or download a PDF |
| Reviewing submissions | It is hard to see what still needs approval | Review and approve data in one place |
